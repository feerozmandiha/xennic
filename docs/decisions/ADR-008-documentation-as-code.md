# ADR-008 Documentation as Code

**نسخه**: ۱.۰.۰ | **وضعیت**: Accepted | **تاریخ**: خرداد ۱۴۰۵

---

## Problem

مستندات پلتفرم به مرور از کد فاصله می‌گیرد (Documentation Drift).

نیاز به مکانیزمی داریم که مستندات همواره با کد همگام باشند.

## Decision

مستندات به عنوان بخشی از کد (Documentation as Code) در نظر گرفته می‌شوند:

1. **هممجوری**: مستندات در همان مخزن (`/docs`) نگهداری می‌شوند
2. **قرارداد**: تغییر کد → تغییر مستندات (قانون الزامی برای PRها)
3. **بازبینی**: هر PR باید شامل بازبینی مستندات باشد
4. **نسخه‌بندی**: مستندات با Semantic Versioning کد همگام هستند
5. **ابزار**: اسناد Markdown + Mermaid diagrams

## Alternatives

| گزینه | مزایا | معایب |
|-------|-------|-------|
| مستندات مجزا (GitHub Wiki) | جداسازی کامل | همگام‌سازی دستی |
| مستندات در کد (Docs as Code) | همگام‌سازی خودکار | حجم مخزن بیشتر |
| سیستم خارجی (Confluence) | ابزار غنی | هزینه + عدم همگام‌سازی |

## Consequences

### Pros
- همگام‌سازی خودکار مستندات با کد
- بازبینی مستندات در PR
- تاریخچه کامل تغییرات مستندات

### Cons
- افزایش حجم مخزن
- نیاز به بازبینی مستندات در هر PR
