## 1. Evidence Node — گره شواهد

واحد اتمی شواهد. هر گره یک قطعه شواهد از یک سند منبع را نمایش می‌دهد.

| فیلد | نوع | توضیح | مثال |
|------|-----|-------|------|
| `evidence_id` | UUID | شناسه یکتای جهانی | XEN-EVI-000042 |
| `eko_id` | UUID | ارجاع به شئ دانش والد | XEN-EKO-1405-000042 |
| `content` | Text | محتوای واقعی شواهد (بخشی از متن، گزاره، داده) | "مقاومت عایقی باید ≥ ۱ مگااهم به ازای هر کیلوولت" |
| `source_reference` | Object | سند، بخش، صفحه، خط | { document: "IEC-60364-6", section: "6.4", page: 12, line: 34 } |
| `source_tier` | Integer (1-5) | تایر منبع | 1 |
| `confidence` | Float (0.0-1.0) | امتیاز اطمینان | 0.95 |
| `evidence_type` | Enum | نوع شواهد | standard_clause |
| `extracted_by` | String | خط لوله یا پردازشگر استخراج‌کننده | k2.0-acquisition-v2.1 |
| `extracted_at` | ISO8601 | زمان استخراج | 2026-06-20T09:30:00Z |
| `checksum` | SHA-256 | هش محتوای شواهد | a3f5b8c1d2e4... |
| `status` | Enum | وضعیت گره | active |

### مقادیر Evidence Type — مقادیر نوع شواهد

| مقدار | توضیح |
|-------|-------|
| `standard_clause` | بند استاندارد رسمی (IEC, IEEE, BS, DIN) |
| `regulation_article` | ماده قانونی یا آیین‌نامه |
| `manufacturer_spec` | مشخصات فنی سازنده تجهیزات |
| `calculation_result` | نتیجه محاسبات مهندسی |
| `measurement` | داده اندازه‌گیری یا تست میدانی |
| `expert_opinion` | نظر کارشناسی |

## 2. Evidence Chains — زنجیره‌های شواهد

یک گراف جهت‌دار غیرمدور (DAG) از گره‌های شواهد که با گام‌های استدلال به یکدیگر متصل شده‌اند.

### ویژگی‌های زنجیره

| فیلد | نوع | توضیح |
|------|-----|-------|
| `chain_id` | UUID | شناسه یکتای زنجیره |
| `eko_ids` | UUID[] | ارجاع به EKOهای استفاده‌شده در زنجیره |
| `nodes` | EvidenceNode[] | آرایه مرتب گره‌های شواهد |
| `edges` | ReasoningEdge[] | گام‌های استدلال اتصال‌دهنده گره‌ها |
| `root_question` | Text | پرسشی که زنجیره را آغاز کرده است |
| `final_conclusion` | EvidenceNode | گره نتیجه نهایی |
| `aggregated_confidence` | Float | اطمینان تجمیعی زنجیره |

### قواعد زنجیره

- هر زنجیره دقیقاً یک ریشه (پرسش) و یک برگ (نتیجه) دارد
- زنجیره باید غیرمدور باشد
- اطمینان = کمترین(m确信 همه گره‌ها) × میانگین(وزن همه یال‌ها)
- اگر هر گرهی اطمینان کمتر از ۰.۳ داشته باشد، کل زنجیره پرچم‌گذاری می‌شود

---

## 3. Evidence Graphs — گراف شواهد

چندین زنجیره شواهد می‌توانند در یک گراف شواهد ادغام شوند هنگام استفاده از چند EKO.

| فیلد | نوع | توضیح |
|------|-----|-------|
| `graph_id` | UUID | شناسه یکتای گراف |
| `chains` | UUID[] | آرایه شناسه زنجیره‌ها |
| `merged_nodes` | EvidenceNode[] | گره‌های اشتراکی میان زنجیره‌ها |
| `consensus_score` | Float | تعداد زنجیره‌های پشتیبان هر نتیجه |
| `conflict_flag` | Boolean | وجود نتایج متناقض میان زنجیره‌ها |

---

## 4. Evidence Aggregation — تجمیع شواهد

| نوع تجمیع | منطق | فرمول اطمینان |
|-----------|------|---------------|
| **Conjunctive** — پیوندی | همه قطعات باید درست باشند | min(C₁, C₂, ..., Cₙ) |
| **Disjunctive** — گسستی | هر قطعه می‌تواند درست باشد | max(C₁, C₂, ..., Cₙ) |
| **Weighted** — وزنی | میانگین وزنی بر اساس تایر منبع | Σ(Wᵢ × Cᵢ) / ΣWᵢ |

## 5. Evidence Ranking — رتبه‌بندی شواهد

| اولویت | معیار | توضیح |
|--------|-------|-------|
| ۱ | **Source Tier** — تایر منبع | منبع معتبرتر رتبه بالاتر |
| ۲ | **Confidence Score** — امتیاز اطمینان | اطمینان بالاتر رتبه بالاتر |
| ۳ | **Temporal Freshness** — تازگی زمانی | شواهد جدیدتر رتبه بالاتر |
| ۴ | **Directness** — مستقیم‌بودن | شواهد مستقیم بر شواهد استنتاجی مقدم است |

---

## 6. Evidence Traceability — ردیابی شواهد

- هر گره شواهد به exactly یک سند منبع ردیابی می‌شود
- هر زنجیره شواهد به پرسش مبدأ ردیابی می‌شود
- منشأ کامل: خط لوله، پردازشگر، مدل ایجادکننده هر گره
- مسیر حسابرسی: تمام تغییرات ثبت می‌شوند

---

## 7. Evidence Expiration — انقضای شواهد

| نوع شواهد | مدت اعتبار | رفتار پس از انقضا |
|-----------|-----------|-------------------|
| استانداردها | تا زمان جایگزینی استاندارد | پرچم‌گذاری و عدم استفاده در استدلال |
| مقررات و آیین‌نامه‌ها | تا زمان تغییر مقررات | پرچم‌گذاری و عدم استفاده در استدلال |
| داده‌های سازنده | ۵ سال | پرچم‌گذاری و عدم استفاده در استدلال |
| دانش عمومی/اجتماعی | ۲ سال | پرچم‌گذاری و عدم استفاده در استدلال |

شواهد منقضی شده: پرچم‌گذاری می‌شوند، در استدلال استفاده نمی‌شوند، برای حسابرسی نگهداری می‌شوند.

## 8. Evidence Versioning — نسخه‌بندی شواهد

- گره‌های شواهد پس از ایجاد تغییرناپذیر (immutable) هستند
- اصلاحات گره شواهد جدید ایجاد می‌کنند (نه تغییر در گره موجود)
- گره‌های شواهد منسوخ به جایگزین خود پیوند می‌خورند
- میدان `superseded_by` در گره منسوخ مقداردهی می‌شود

## 9. Evidence Confidence — اطمینان شواهد

| مؤلفه | توضیح | دامنه |
|-------|-------|-------|
| **Source Confidence** — اطمینان منبع | بر اساس تایر منبع (مطابق confidence-scoring.md) | ۰.۰ — ۱.۰ |
| **Extraction Confidence** — اطمینان استخراج | میزان اطمینان از صحت استخراج شواهد | ۰.۰ — ۱.۰ |
| **Relevance Confidence** — اطمینان مرتبطی | میزان مرتبطی شواهد با پرسش | ۰.۰ — ۱.۰ |
| **Combined** — ترکیبی | حاصلضرب هر سه مؤلفه | ۰.۰ — ۱.۰ |

```
Combined = Source × Extraction × Relevance
```

## 10. Evidence Inheritance — وراثت شواهد

- اگر EKO A از منبع S مشتق شده باشد، شواهد S به A به ارث می‌رسد
- وراثت `source_tier` و `extraction_confidence` را حفظ می‌کند
- عمق زنجیره وراثت محدود به ۳ هاپ است
- `inherited_from` در گره شواهد ثبت می‌شود

## 12. AI & Graph RAG Compatibility — سازگاری با هوش مصنوعی و گراف RAG

| مفهوم شواهد | معادل در گراف دانش |
|-------------|-------------------|
| Evidence Node | گره گراف با برچسب `Evidence` |
| Evidence Chain | زیرگراف با یال‌های `REASONING_EDGE` |
| Evidence Graph | گراف تجمیعی با یال‌های `HAS_EVIDENCE` |
| رابطه پشتیبانی | یال `SUPPORTS` با وزن اطمینان |
| رابطه تضاد | یال `CONTRADICTS` با وزن تضاد |

### Embedding و بازیابی

- محتوای شواهد برای جستجوی شباهت (similarity search) embedding می‌شود
- از embedding برداری برای یافتن گره‌های شواهد مرتبط با پرسش استفاده می‌شود
- گراف شواهد برای استدلال چند-hop traversal می‌شود
- نتایج جستجوی برداری با traversal گراف ترکیب می‌شوند
