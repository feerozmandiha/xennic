# رانتایم بررسی انسانی — Human Review Runtime

**Version:** 1.0.0 | **Status:** Draft | **Last Updated:** Tir 1405

---

## Purpose — هدف

The Human Review Runtime manages escalation of AI reasoning conclusions to qualified human engineers when automated confidence, evidence integrity, or safety thresholds cannot be met.

---

## Escalation Triggers — محرک‌های ارجاع

| # | محرک | شرط |
|---|------|------|
| 1 | **Low Confidence** — اطمینان پایین | اطمینان نهایی < ۰٫۳ |
| 2 | **Conflicting Tier-1 Sources** — تضاد منابع رده یک | دو یا چند استاندارد Tier 1 الزامات متضاد دارند |
| 3 | **Safety-Critical Decisions** — تصمیمات حیاتی ایمنی | پرسش شامل: تنظیمات حفاظتی، حدود جریان خطا، طراحی ارت، فاصله‌های ایمنی، سیستم‌های اضطراری |
| 4 | **Incomplete Evidence** — گواه ناقص | زنجیره گواه دارای شکاف است (اطمینان < ۰٫۵ در هر گام) |
| 5 | **Regulatory Conflicts** — تضاد نظارتی | مقرره با استاندارد یا مقرره دیگر تضاد دارد |
| 6 | **Unknown Domain** — دامنه نامشخص | دامنه پرسش قابل تعیین نیست |
| 7 | **Novel Query** — پرسش بدیع | هیچ پرسش مشابهی در تاریخچه وجود ندارد (اولین‌ازنوع) |
| 8 | **User Request** — درخواست کاربر | کاربر صریحاً درخواست بررسی انسانی می‌کند |

## Human Review Workflow — گردش کار بررسی انسانی

| گام | شرح |
|-----|------|
| **1. Escalation** — ارجاع | Reasoning runtime بسته‌ای شامل: پرسش، زمینه، نتیجه پیشنهادی، زنجیره گواه، تفکیک اطمینان، جزئیات تضاد را می‌سازد |
| **2. Queue** — صف | آیتم ارجاعی با اولویت (critical / high / normal) وارد صف بررسی انسانی می‌شود |
| **3. Assignment** — تخصیص | بر اساس دامنه به بازبین واجد شرایط تخصیص می‌یابد |
| **4. Review** — بررسی | بازبین: گواه، استدلال، اطمینان، گزینه‌های جایگزین را بررسی می‌کند |
| **5. Decision** — تصمیم | یکی از: Approve (تأیید)، Reject (رد)، Modify (اصلاح)، Request More Evidence (درخواست گواه بیشتر) |
| **6. Feedback** — بازخورد | بازبین توجیه تصمیم خود را ارائه می‌دهد |
| **7. Resolution** — نهایی‌سازی | |
| | Approve → نتیجه عیناً تحویل می‌شود |
| | Reject → کاربر با توضیح مطلع می‌شود |
| | Modify → نتیجه اصلاح‌شده با تغییرات بازبین تحویل می‌شود |
| | Request More → AI گواه اضافی جمع‌آوری و دوباره ارجاع می‌دهد |
| **8. Learning** — یادگیری | تصمیمات تأییدشده انسانی به عنوان سابقه برای پرسش‌های مشابه آینده ذخیره می‌شوند |

## Review Dashboard — داشبورد بررسی

| ویژگی | توضیح |
|-------|-------|
| **Sorting** — مرتب‌سازی | اولویت، دامنه، زمان انتظار |

| **Reviewer Tools** — ابزار بازبین | مقایسه هم‌زمان منابع، نمایش گراف زنجیره گواه، اجرای بررسی فرمول |
| **SLA Monitoring** — نظارت SLA | عادی ۴ ساعت، بالا ۱ ساعت، بحرانی ۳۰ دقیقه |

## SLA and Escalation — SLA و ارجاع مجدد

| اولویت | زمان هدف | ارجاع در صورت تأخیر |
|--------|----------|---------------------|
| **Critical** — بحرانی | ۳۰ دقیقه | اطلاع به بازبین ارشد + سرپرست مهندسی |
| **High** — بالا | ۱ ساعت | اطلاع به سرپرست دامنه |
| **Normal** — عادی | ۴ ساعت | اطلاع به بازبین |
| **Low** — پایین | ۲۴ ساعت |なし (بررسی پس‌زمینه) |

## Review Audit — حسابرسی بررسی

| مورد | توضیح |
|------|-------|
| **Logging** — ثبت | هر اقدام بررسی ثبت می‌شود: reviewer_id، زمان، تصمیم، توجیه |
| **History** — تاریخچه | تاریخچه بررسی برای: حسابرسی کیفیت، عملکرد بازبین، استخراج سابقه |
| **Monthly Report** — گزارش ماهانه | نرخ تأیید، نرخ بازگشت، میانگین زمان بررسی |
