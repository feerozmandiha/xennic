# رانتایم حقیقت مهندسی — Engineering Truth Runtime

**Version:** 1.0.0 | **Status:** Draft | **Last Updated:** Tir 1405

---

## Truth Model — مدل حقیقت

The Engineering Truth Runtime operationalises the K1.5 Truth Model as a validation layer that ensures every AI engineering conclusion is sound across all five truth elements:

| عنصر | توضیح |
|------|-------|
| **Source** | منبع ادعا (EKO یا مرجع استاندارد) |
| **Evidence** | شواهد مستخرج از منبع |
| **Reasoning** | مسیر استدلال از شواهد تا نتیجه |
| **Confidence** | امتیاز اطمینان ترکیبی |
| **Version** | نسخه تمام اجزای دخیل در استنتاج |

## Truth Validation Per Element — اعتبارسنجی حقیقت به تفکیک عنصر

### 1. Source Validation — اعتبارسنجی منبع

| شرط | رفتار در نقض |
|-----|-------------|
| هر ادعا باید حداقل به یک منبع ارجاع دهد | رد ادعا، پرچم «unsupported statement» |
| منبع باید در Knowledge Base موجود باشد (eko_id معتبر یا مرجع استاندارد) | رد ادعا، پرچم «invalid source reference» |
| منبع باید در مرحله صحیح چرخه حیات باشد (Published، نه Archived) | رد ادعا، پرچم «source lifecycle mismatch» |
| منبع باید در بازه زمانی اعتبار خود باشد | رد ادعا، پرچم «source expired» |
| رتبه منبع باید مطابق source-hierarchy.md طبقه‌بندی شده باشد | رد ادعا، پرچم «untiered source» |

### 2. Evidence Validation — اعتبارسنجی شواهد

| شرط | رفتار در نقض |
|-----|-------------|
| هر ادعا باید با گواه مستخرج از منبع پشتیبانی شود | رد ادعا، پرچم «hallucination» |
| گواه باید قابل استخراج از بند/قسمت مشخص منبع باشد | رد ادعا، پرچم «evidence not found» |
| اطمینان گواه باید حداقل آستانه را داشته باشد (Tier 1-2 ≥ 0.5، Tier 3-5 ≥ 0.3) | کاهش رتبه ادعا، پرچم «low evidence confidence» |
| زنجیره گواه باید بدون دور باشد | رد ادعا، پرچم «circular evidence» |

### 3. Reasoning Validation — اعتبارسنجی استدلال

| شرط | رفتار در نقض |
|-----|-------------|
| مسیر استدلال باید مستند باشد (توالی گام‌ها) | رد ادعا، پرچم «missing reasoning path» |
| هر گام باید به یک حالت استدلال ارجاع دهد (قیاسی/استقرایی و غیره) | رد ادعا، پرچم «untyped reasoning step» |
| هر گام باید حداقل یک ارجاع گواه داشته باشد | رد ادعا، پرچم «unsupported reasoning step» |
| مسیر استدلال باید غیرچرخشی باشد | رد ادعا، پرچم «circular reasoning» |
| گام‌های استدلال باید منطقاً منسجم باشند (مقدمه → نتیجه) | رد ادعا، پرچم «invalid reasoning» |

### 4. Confidence Validation — اعتبارسنجی اطمینان

| شرط | رفتار در نقض |
|-----|-------------|
| امتیاز اطمینان باید طبق فرمول confidence-engine.md محاسبه شود | محاسبه مجدد |
| هر ۵ مؤلفه اطمینان باید مقداردهی شده باشند | محاسبه مجدد با مقدار پیش‌فرض |
| امتیاز باید طبق جدول تفسیر امتیاز قابل تفسیر باشد | پرچم «uninterpretable confidence» |
| اگر اطمینان < ۰٫۳ باشد، باید به انسان ارجاع داده شود | ارجاع اجباری به Human Review |

**Score Interpretation Table — جدول تفسیر امتیاز**

| محدوده اطمینان | تفسیر | اقدام |
|----------------|--------|-------|
| ≥ ۰٫۹ | Very High — بسیار بالا | تحویل مستقیم |
| ۰٫۷ – ۰٫۸۹ | High — بالا | تحویل مستقیم |
| ۰٫۵ – ۰٫۶۹ | Moderate — متوسط | تحویل با هشدار |
| ۰٫۳ – ۰٫۴۹ | Low — پایین | تحویل با هشدار جدی |
| < ۰٫۳ | Very Low — بسیار پایین | ارجاع اجباری به انسان |

### 5. Version Validation — اعتبارسنجی نسخه

| شرط | رفتار در نقض |
|-----|-------------|
| تمام EKOهای استفاده شده باید نسخه ثبت شده داشته باشند | پرچم «unversioned EKO»، اعمال حداقل اطمینان |
| تمام فرمول‌های استفاده شده باید نسخه ثبت شده داشته باشند | پرچم «unversioned formula»، اعمال حداقل اطمینان |
| تمام قوانین استفاده شده باید نسخه ثبت شده داشته باشند | پرچم «unversioned rule»، اعمال حداقل اطمینان |
| اگر هر نسخه‌ای منسوخ شده باشد، پرچم تازگی زده شود | پرچم «freshness check required» |

## Truth Runtime Architecture — معماری رانتایم حقیقت

| مؤلفه | نقش | زمان اجرا |
|-------|-----|-----------|
| **Truth Validator Service** | اعتبارسنجی هر ۵ عنصر حقیقت پیش از تحویل نتیجه | پیش از تحویل به کاربر |
| **Truth Checker** | تعبیه‌شده در reasoning runtime (گام ۷٫۵، میان ارزیابی فرمول و محاسبه اطمینان) | درون خط لوله استدلال |
| **Truth Auditor** | بررسی دوره‌ای پیشینه نتایج موجود | پس‌زمینه |
| **Truth Cache** | حافظه نهان اعتبارسنجی‌ها (باطل‌شونده با تغییر منبع/گواه) | همیشه |

## Truth Enforcement Levels — سطوح اجرای حقیقت

| سطح | رفتار | زمان استفاده |
|-----|-------|-------------|
| **Strict** — سختگیرانه | مسدود کردن هر نتیجه缺失 هر عنصر | Production (پیش‌فرض) |
| **Moderate** — میانه | هشدار اما اجازه تحویل با پرچم | توسعه، استفاده داخلی |
| **Relaxed** — سهل‌گیرانه | فقط ثبت در لاگ | تست، اشکال‌زدایی |

## Truth Violation Severity — شدت نقض حقیقت

| شدت | شرط | اقدام |
|-----|-----|-------|
| **CRITICAL** | بدون منبع، بدون گواه، یا اطمینان < ۰٫۳ | مسدود کردن نتیجه، ارجاع به انسان |
| **HIGH** | گواه ناقص، شکاف استدلالی | پرچم نتیجه، غیرقابل استفاده برای طراحی |
| **MEDIUM** | نسخه منسوخ، اطمینان ۰٫۳ – ۰٫۵ | تحویل با هشدار |
| **LOW** | قالب‌بندی جزئی، استناد ناقص | تحویل با یادداشت |
