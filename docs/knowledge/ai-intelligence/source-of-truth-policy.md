# خط‌مشی منبع واحد حقیقت — Source of Truth Policy

**Version:** 1.0.0 | **Status:** Draft | **Last Updated:** Tir 1405

---

## Core Principles — اصول بنیادین

### 1. Primary Source Rule — قانون منبع اولیه

هر ادعا باید به یک منبع در پایگاه دانش قابل ردیابی باشد. اگر منبع از نوع Tier 1 (استاندارد بین‌المللی) باشد، پاسخ معتبر تلقی می‌شود. اگر هیچ منبعی وجود نداشته باشد، AI باید صریحاً بیان کند که اطلاعات در پایگاه دانش موجود نیست.

| وضعیت منبع | رفتار AI |
|-----------|---------|
| منبع Tier 1 موجود | پاسخ معتبر — ارجاع مستقیم به استاندارد |
| منبع Tier 2-3 موجود | پاسخ با درجه اطمینان پایین‌تر — ذکر تایر منبع |
| منبع Tier 4-5 موجود | پاسخ با هشدار کیفیت منبع |
| هیچ منبعی موجود نیست | اعلام صریح "این اطلاعات در پایگاه دانش موجود نیست" |

### 2. No Speculation — ممنوعیت حدس و گمان

AI نمی‌تواند نتایج مهندسی را بدون شواهد تولید کند. در صورت فقدان اطلاعات، AI باید از عبارات زیر استفاده کند:

| سناریو | عبارت اجباری |
|--------|-------------|
| اطلاعات وجود ندارد | "This information is not available in the current knowledge base." |
| اطلاعات ناقص است | "The knowledge base contains partial information on this topic. The following aspects are not covered: [...]" |
| اطلاعات نیاز به تأیید دارد | "This information could not be verified against authoritative sources. Manual verification is required." |

### 3. Source Precedence — تقدم منابع

هنگامی که چندین منبع وجود دارد، ترتیب اولویت به شرح زیر است:

```
Tier 1 (استاندارد بین‌المللی) > Tier 2 (استاندارد ملی) > Tier 3 (کاتالوگ رسمی) > Tier 4 (منابع معتبر) > Tier 5 (محتوای تولیدی)
```

| معیار تقدم | توضیح | مثال |
|-----------|-------|------|
| **تایر بالاتر** | منبع با تایر بالاتر مقدم است | IEC 60909 (Tier 1) > ISIRI 1234 (Tier 2) |
| **تاریخ جدیدتر** | در تایر یکسان، منبع جدیدتر مقدم است | IEC 60364:2022 > IEC 60364:2018 |
| **تخصصی‌تر** | در تایر و تاریخ یکسان، منبع خاص‌تر مقدم است | IEC 60204-1 (ماشین‌آلات) > IEC 60364 (تأسیسات عمومی) |

### 4. Conflicting Sources Protocol — پروتکل منابع متضاد

هنگامی که منابع با یکدیگر تضاد دارند:

```
Step 1: شناسایی تضاد
        └── تشخیص اختلاف در مقادیر عددی، پارامترهای طراحی یا الزامات

Step 2: تحلیل تایر
        ├── اگر تایرها متفاوت → منبع با تایر بالاتر مقدم است
        └── اگر تایرها یکسان → نیاز به بررسی بیشتر

Step 3: گزارش
        ├── ذکر صریح وجود تضاد
        ├── ارائه هر دو دیدگاه همراه با منابع
        ├── اعلام توصیه بر اساس تایر
        └── پرچم‌گذاری برای بازبینی کارشناس انسانی
```

| نوع تضاد | پروتکل |
|---------|--------|
| Tier 1 vs Tier 2 | منبع Tier 1 مقدم — ذکر تفاوت با Tier 2 |
| Tier 1 vs Tier 1 (نسخه‌های مختلف) | نسخه جدیدتر مقدم — ذکر تغییرات |
| Tier 1 vs Tier 1 (استانداردهای مختلف) | استاندارد خاص‌تر مقدم — پرچم برای کارشناس |
| Tier 2 vs Tier 2 (مراجع ملی مختلف) | پرچم قرمز — نیاز به تصمیم کارشناسی |
| Tier 3 vs Tier 3 (کاتالوگ‌های مختلف) | ارائه هر دو — انتخاب بر عهده کاربر |

### 5. Unavailable Information Protocol — پروتکل اطلاعات در دسترس

| دسته | مثال | رفتار AI |
|------|------|---------|
| **کاملاً缺失** | استانداردی که هنوز ایندکس نشده | "This standard is not available in the knowledge base." |
| **ناقص** | بخشی از استاندارد موجود است | پاسخ با ذکر محدوده موجود و بخش‌های缺失 |
| **قدیمی** | نسخه منسوخ استاندارد موجود است | پاسخ با هشدار "This standard version may be superseded" |
| **محدود** | منبع Tier 4-5 موجود است | پاسخ با هشدار کیفیت و توصیه به تأیید |
| **محرمانه** | اطلاعات وجود دارد اما کاربر دسترسی ندارد | "You do not have access to the required source documents." |

## Enforcement — اجرا و نظارت

| مکانیزم | توضیح | مسئول |
|---------|-------|-------|
| **Pre-generation Validation** | بررسی وجود منبع قبل از تولید پاسخ | Source Validator |
| **Post-generation Audit** | تطابق خودکار ادعاها با منابع | Hallucination Detector |
| **Confidence Gate** | مسدودسازی پاسخ‌های با اطمینان پایین | Confidence Scorer |
| **Human Review Queue** | ارسال موارد پرچم‌شده به کارشناس | AI Service |
| **Policy Violation Log** | ثبت تمام موارد نقض خط‌مشی | Audit System |

| سطح نقض | پیامد |
|---------|-------|
| **Minor** — ارجاع ناقص اما محتوای صحیح | اصلاح خودکار + لاگ |
| **Moderate** — ادعای بدون منبع | مسدودسازی پاسخ + بازبینی انسانی |
| **Critical** — اطلاعات نادرست مهندسی | مسدودسازی + بررسی فوری + به‌روزرسانی خط‌مشی |

## Compliance Matrix — ماتریس انطباق

| شرط | الزام | روش تأیید |
|-----|-------|----------|
| هر پاسخ حداقل یک منبع دارد | ✅ اجباری | Source Validator |
| پاسخ‌های با اطمینان < 0.5 مسدود می‌شوند | ✅ اجباری | Confidence Gate |
| منابع متضاد گزارش می‌شوند | ✅ اجباری | Conflict Detector |
| اطلاعات缺失 صریحاً اعلام می‌شود | ✅ اجباری | Response Auditor |
| پاسخ‌های مهندسی قبل از تحویل اعتبارسنجی می‌شوند | ✅ اجباری | Validation Pipeline |

> این خط‌مشی برای تمام سرویس‌های مصرف‌کننده دانش (AI Service, Engineering Service, API) الزامی است.
